Aqui estão os casos de uso principais, desenhados para serem escaláveis:1. Gestor (Admin da Empresa)UC01: Criação de Campanha de IncentivoAtor: Admin da Empresa.Fluxo: O Admin acessa o painel, define um nome para a missão (ex: "Venda de Verão"), o valor em pontos, e o critério de conclusão. Ele pode restringir a missão a um departamento específico (ex: "Vendas").Regra de Ouro: O sistema deve validar se o company_id da missão corresponde ao company_id do Admin logado.UC02: Aprovação de Resgate de PrêmiosAtor: Admin da Empresa / RH.Fluxo: O Admin recebe uma notificação de que o "Colaborador X" resgatou um "Voucher de Café". O Admin verifica o estoque físico e clica em "Marcar como Entregue".Ponto de Escala: O sistema deve gerar um código de resgate único (QR Code ou Hash) para evitar fraudes.2. Colaborador (Usuário Final)UC03: Realização de Missão e Ganho de PontosAtor: Colaborador.Fluxo: O colaborador visualiza as missões disponíveis. Ao concluir uma (seja por input manual do admin ou integração via API), seu saldo é atualizado instantaneamente.Feedback Visual: Disparo de um "Toast" ou animação de confetes na tela (UX Neon Noir).UC04: Resgate de Recompensa na LojaAtor: Colaborador.Fluxo: O usuário entra na loja, escolhe um prêmio de 500 pontos. O sistema verifica se ele tem saldo. Se sim, bloqueia os pontos (status: reserved), cria o pedido e subtrai do saldo total.Regra de Negócio: Se o estoque for 0, o botão de resgate deve ser desabilitado automaticamente.3. Sistema (Multi-tenant & Segurança)UC05: Isolamento Total de Dados (The "Wall")Ator: Sistema.Cenário: O usuário da Empresa A tenta acessar a URL app.com/api/v1/missions/500, mas a missão 500 pertence à Empresa B.Comportamento: O sistema deve retornar um erro 404 (Not Found) e não um 403 (Forbidden). Isso evita que o usuário saiba que aquele ID de fato existe em outra empresa (técnica de Security by Obscurity).UC06: Upgrade de Plano (Upsell)Ator: Super Admin (Você).Fluxo: Quando uma empresa atinge o limite de 50 colaboradores (plano free), o sistema bloqueia novos cadastros e exibe um modal de upgrade para o plano Pro.Escalabilidade: Isso garante que o seu banco de dados não cresça indefinidamente sem faturamento.Matriz de Rastreabilidade (Simples)Caso de UsoTabela PrincipalImpacto na EscalaUC01 (Missão)missionsBaixo (Escrita simples)UC03 (Ganhos)point_transactionsAlto (Muitas inserções - requer indexação)UC04 (Loja)orders / rewardsMédio (Requer controle de concorrência/estoque)UC05 (Isolamento)companiesCrítico (Segurança jurídica do SaaS)Exemplo de Fluxo Jira para o UC04 (Resgate):Task 1: Criar API endpoint POST /orders.Task 2: Implementar Database Transaction (Se o débito de pontos falhar, o pedido não pode ser criado).Task 3: Criar trigger de e-mail para o RH notificando o novo resgate.Task 4: Frontend: Componente de feedback de "Sucesso" com o código do voucher.
