PRD & Implementation Roadmap — Gamifica SaaS (Multi-tenant)OverviewPlataforma SaaS onde múltiplas empresas podem criar programas de incentivo. Cada empresa tem seus colaboradores, missões e loja de prêmios. O isolamento de dados é garantido via company_id.Tecnologia Sugerida: Laravel (PHP), Livewire, Tailwind CSS.Fase 1: Fundação Multi-tenant & AuthFoco: Garantir que o "Prédio" suporte vários "Apartamentos" (Empresas) isolados.1.1: Database Schema & Tenant isolation[ ] Criar migração companies: id, name, slug (unique), logo, plan_type (basic/pro).[ ] Atualizar migração users: adicionar company_id (FK), role (admin/employee), points_balance.[ ] Middleware Global: Criar TenantMiddleware que identifica a empresa pela URL ou pelo login e aplica um escopo global: static::addGlobalScope('company', function (Builder $builder) { $builder->where('company_id', auth()->user()->company_id); });1.2: Autenticação de Dois Níveis[ ] Login Admin: Acesso ao painel de gestão (missões/loja).[ ] Login Colaborador: Acesso à área de jogos e recompensas.Tests (tests/Feature/Auth/TenantAuthTest.php):[ ] Garantir que Admin da Empresa A não consiga listar usuários da Empresa B.[ ] Testar se o registro de novo usuário vincula automaticamente ao company_id correto.Fase 2: Gestão de Gamificação (Painel Admin)Onde o RH configura as regras do jogo.2.1: Missões e Atividades[ ] Criar model Mission: id, company_id, title, description, points_reward, is_active.[ ] CRUD de Missões com validação de company_id.2.2: Loja de Prêmios (Rewards)[ ] Criar model Reward: id, company_id, title, cost, stock_quantity.[ ] Interface de upload de fotos dos prêmios.Tests (tests/Feature/Admin/MissionTest.php):[ ] Testar se uma missão criada pela Empresa A aparece apenas para usuários da Empresa A.[ ] Validar que pontos de recompensa não podem ser negativos.Fase 3: Jornada do Colaborador (Player UX)A interface lúdica para quem ganha os pontos.3.1: Dashboard do Player[ ] Componente Livewire para exibir saldo de pontos em tempo real.[ ] Lista de "Missões Disponíveis" filtradas por company_id.3.2: Fluxo de Resgate (Redemption)[ ] Criar model Order: id, user_id, reward_id, status (pending/delivered).[ ] Service de Transação: Criar PointsService para subtrair pontos e gerar pedido em uma única transação de banco (atomicidade).Tests (tests/Feature/Player/RedemptionTest.php):[ ] Testar resgate de prêmio com saldo insuficiente (deve falhar).[ ] Testar resgate com saldo suficiente (deve debitar e criar pedido).Fase 4: Escalabilidade & SocialFuncionalidades que tornam o sistema robusto e viciante.4.1: Ranking (Leaderboard) Multi-tenant[ ] Criar consulta que agrupa usuários da mesma empresa por saldo de pontos.[ ] Cachear o ranking (Redis) por company_id para evitar lentidão com muitos usuários.4.2: Feed de Conquistas[ ] Criar model ActivityLog: id, company_id, message (ex: "João completou a missão X").[ ] Componente de scroll infinito para o feed.Tests (tests/Feature/Social/LeaderboardTest.php):[ ] Garantir que o ranking não misture usuários de empresas diferentes.Estrutura de Tabelas (Visão Jira)TabelaColuna
